Dziennik API

Trzy endpointy do pracy z dziennikiem programowo: dodaj wpis, zapytaj agenta, odczytaj dzień. Per-user, autoryzowane długoterminowym tokenem.

Base URL

base 
https://kkegymepatwufnemtldr.supabase.co/functions/v1

Quick start

curl 
curl -X POST $URL/add-entry \
  -H "Authorization: Bearer dzn_xxx" \
  -H "Content-Type: application/json" \
  -d '{"content":"hello"}'
Wszystkie endpointy są edge functions wdrożone na Supabase. Daty domyślne (date nie podane) używają strefy Europe/Warsaw.

Autoryzacja

Każde żądanie musi zawierać nagłówek Authorization z tokenem API. Tokeny mają format dzn_<base64url> i są związane z jednym kontem użytkownika.

header 
Authorization: Bearer dzn_<twoj_token>

Serwer akceptuje też Supabase JWT (używane przez frontend) — token API jest dla integracji i automatyzacji.

Token wygenerujesz przyciskiem „Generuj token API" w górnym pasku (wymaga zalogowania). Plaintext widoczny jest tylko raz, w momencie wygenerowania.

Błędy & limity

Wszystkie błędy zwracają JSON w formacie { "error": "..." } z odpowiednim kodem HTTP.

StatusZnaczenie
400Niepoprawne body lub parametry (np. zły format daty, content poza 1–500 znakami).
401Brak nagłówka Authorization, zły token, lub token cofnięty.
405Zła metoda HTTP.
409Konflikt — dla Create oznacza osiągnięty limit 5 wpisów na dzień.
500Błąd po stronie serwera.
502Tylko Ask — model AI nieosiągalny.

Limit twardy: 5 wpisów na dzień kalendarzowy per user.

Create — dodaj wpis

POST /add-entry

Tworzy nowy wpis dla autoryzowanego użytkownika. Pozycja (1–5) przydzielana automatycznie. Opcjonalnie zapisuje też mood 1–5 dla danego dnia.

Body

FieldTypeOpis
contentrequiredstringTreść wpisu, 1–500 znaków po trim().
dateoptionalstring (YYYY-MM-DD)Domyślnie dzisiaj w strefie Europe/Warsaw.
moodoptionalinteger 1–5Nastrój dnia. Upsert do day_moods.

Request

curl 
curl -X POST $URL/add-entry \
  -H "Authorization: Bearer dzn_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "dobry dzien, zbudowalem skilla",
    "mood": 5
  }'

Response

200 OKWpis utworzony
json
{
  "id": "d66aa9dd-...",
  "date": "2026-06-23",
  "position": 2,
  "content": "dobry dzien, zbudowalem skilla",
  "mood": 5
}
409 ConflictLimit 5 wpisów na dzień
json
{
  "error": "Limit 5 wpisow na dzien osiagniety.",
  "date": "2026-06-23",
  "count": 5
}

Ask — zapytaj agenta

POST /notes-chat

Pytanie do agenta (Albert Einstein, model grok-3). Agent ma tool-call getNotesForRange i czyta Twoje wpisy z dowolnego zakresu dat. Wpisy z dnia date są w kontekście automatycznie.

Body (single-shot)

FieldTypeOpis
questionrequiredstringPojedyncze pytanie.
dateoptionalstring (YYYY-MM-DD)„Aktywny" dzień — wpisy trafiają do kontekstu. Domyślnie dziś.
Endpoint akceptuje też { messages, context } używany przez UI z historią konwersacji.

Request

curl 
curl -X POST $URL/notes-chat \
  -H "Authorization: Bearer dzn_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "co robilem w zeszly piatek?",
    "date": "2026-06-19"
  }'

Response

200 OK
json
{
  "reply": "W piątek 2026-06-19 zapisałeś dwa wpisy...",
  "proposals": []
}

proposals to lista propozycji nowych wpisów (tool-call proposeNote). Agent nie zapisuje sam — chcesz przyjąć, użyj Create.

Tokeny API

Tokeny przechowywane jako sha256(token) w hex w kolumnie token_hash. Surowy token widoczny tylko w momencie generacji.

Generacja

Kliknij „Generuj token API" w górnym pasku. Musisz być zalogowany w aplikacji.

Cofnięcie tokenu

Aktualnie: update api_tokens set revoked_at = now() where id = '<token_id>'; w Supabase Studio. UI do listy/cofania planowane w v1.1.

Dziennik MCP

Serwer Model Context Protocol hostowany na Vercelu. Pozwala dowolnemu klientowi MCP (Claude Desktop, Claude Code, Cursor, Continue) korzystać z Twojego dziennika jako narzędzia.

Endpoint

url 
https://<twoj-vercel-app>/api/mcp
Endpoint jest stateless (HTTP JSON-RPC). Autoryzacja przez nagłówek Authorization: Bearer dzn_<token>. Token wygenerujesz przyciskiem w górnym pasku.

Endpoint & auth

Wszystkie wywołania trafiają na POST /api/mcp z body JSON-RPC 2.0. Serwer rozumie:

  • initialize — handshake protokołu
  • tools/list — lista dostępnych narzędzi
  • tools/call — wywołanie narzędzia z argumentami
  • ping — health check

Surowy test (curl)

curl 
curl -X POST https://YOUR.vercel.app/api/mcp \
  -H "Authorization: Bearer dzn_xxx" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'

Claude Code

W terminalu, w katalogu projektu:

shell 
claude mcp add --scope user --transport http dziennik \
  https://<twoj-vercel-app>/api/mcp \
  --header "Authorization: Bearer dzn_<twoj_token>"

Restart sesji. Sprawdź /mcp — dziennik powinno mieć status connected.

Claude Desktop

Edytuj claude_desktop_config.json:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json
json 
{
  "mcpServers": {
    "dziennik": {
      "type": "http",
      "url": "https://<twoj-vercel-app>/api/mcp",
      "headers": {
        "Authorization": "Bearer dzn_<twoj_token>"
      }
    }
  }
}

Po edycji zamknij i otwórz Claude Desktop od nowa.

ask

TOOL ask(question, date?)

Pyta asystenta o wpisy. Mapuje na POST /notes-chat. Asystent ma własne tool-calle (czytanie zakresów, propozycje wpisów).

FieldTypeOpis
questionrequiredstringPytanie po polsku.
dateoptionalstring YYYY-MM-DDAktywny dzień (kontekst).
Zwrot zawiera reply (tekst) i opcjonalne proposals. Tool nie tworzy wpisów sam — daj agentowi add_entry, jeśli chcesz, żeby zapisywał.
